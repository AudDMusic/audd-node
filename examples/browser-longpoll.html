AudD browser longpoll demo

Demonstrates the tokenless LongpollConsumer from audd/longpoll — browser-safe, carries no API token. Your server should derive the category server-side via audd.streams.deriveLongpollCategory(radioId) and ship it to the browser.

Boilerplate (paste into your bundler)

// import via your bundler so this stays browser-safe (no API token):
import { LongpollConsumer } from "audd/longpoll";

const consumer = new LongpollConsumer("088719f57");
const poll = consumer.iterate({ timeout: 30 });
for await (const m of poll.matches) {
  console.log(m.song.artist, m.song.title);
}
// notifications and errors are also iterables on `poll`

In an actual app you'd build this with esbuild/vite/rollup. This page is a static reference — it doesn't import the SDK to keep the demo runnable from file://.
